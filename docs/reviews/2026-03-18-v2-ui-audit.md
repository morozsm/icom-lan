## 4. Known Issues

### 🔴 P0: Critical Bugs

#### **Issue #1: FilterPanel Width Slider Broken**
- **Component:** `FilterPanel.svelte`
- **Symptom:** Width slider (50-3600 Hz) calls `set_filter_width` command
- **Problem:** Backend handler missing → `ValueError: unhandled command: 'set_filter_width'`
- **Impact:** User gets error when adjusting filter width
- **Fix:** Implement `set_filter_width` in backend (1-2h)
  - `commands.py`: add builder function
  - `handlers.py`: add case + enqueue
  - `radio_poller.py`: add dataclass + execution
  - Tests: unit + integration

**Related:**
- IF Shift and PBT sliders likely work (map to `set_pbt_inner/outer` which exist)

---

### 🟡 P1: Visual Parity Issues

#### **Issue #2: VFO Layout vs Pencil Mockup (#290-294)**
- **Problem:** Hard-coded pixel values cause composition drift on large screens
- **Specific:** S-meter reads too small on 4K displays
- **Status:** 4 duplicate issues open (#290, #291, #292, #293) + 1 architecture issue (#294)
- **Fix:** Centralize layout tokens, add bounded scaling (2-3h per #294 plan)

**Acceptance criteria from #294:**
- [ ] Centralize top-row VFO layout tokens as CSS custom properties
- [ ] Refactor VfoPanel and VfoOps to consume shared tokens
- [ ] Add VFO-specific LinearSMeter visual variant
- [ ] Add bounded wide-screen token overrides
- [ ] Tests for token usage and meter variant
