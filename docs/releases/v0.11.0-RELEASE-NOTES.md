# icom-lan v0.11.0 Release Notes

**Release Date**: 2026-03-23
**Version**: 0.11.0
**Status**: ✅ Production Ready

## Headline: Multi-Radio Expansion Complete (M5)

v0.11.0 marks the completion of the **Multi-Radio Expansion (M5)** phase, adding support for three new Icom transceiver models alongside IC-7610. All three new models now have complete serial + optional LAN backends with full feature parity and comprehensive documentation.

---

## Major Features

### 🎯 Multi-Radio Backend Architecture (M5)

#### New Models Supported
- **IC-705** (portable HF/VHF/UHF)
  - Serial backend: USB CI-V + USB audio
  - LAN backend: Full network support
  - CI-V address: 0xA4 | Single receiver
  - Status: ✅ Software-ready, hardware validation pending

- **IC-7300** (fixed HF/50 MHz)
  - Serial backend: USB CI-V + USB audio
  - LAN backend: Network only (no USB LAN)
  - CI-V address: 0x94 | Single receiver
  - Status: ✅ Software-ready, hardware validation pending

- **IC-9700** (fixed VHF/UHF satellite)
  - Serial backend: USB CI-V + USB audio
  - LAN backend: Full network support (Ethernet)
  - CI-V address: 0xA2 | **Dual-receiver** (unique!)
  - Status: ✅ Software-ready, hardware validation pending

#### Factory Model Routing
```python
# Auto-detects model and routes to correct backend
radio = IcomRadio(backend="serial", model="IC-9700", device="/dev/cu.usbserial-XXX")

# Or specify explicitly
radio = IcomRadio(
    backend="lan",
    host="192.168.1.100",
    model="IC-705"
)
```

#### Profile-Driven Architecture
- Each model has a `.toml` profile defining CI-V address, capabilities, frequency ranges, filters
- Profiles in `rigs/` directory: `ic705.toml`, `ic7300.toml`, `ic9700.toml`
- Case-insensitive model matching: `"ic-705"`, `"IC-705"`, `"Ic705"` all work
- Zero code duplication: All backends inherit from `Icom7610CoreRadio`

### 📊 Dual-Receiver Support (IC-9700 Unique)

The IC-9700 is the first radio with **independent dual receivers**:

```python
# Monitor both MAIN and SUB simultaneously
main_freq = await radio.get_frequency(receiver=0)  # MAIN
sub_freq = await radio.get_frequency(receiver=1)   # SUB

# Set different frequencies on each
await radio.set_frequency(144_100_000, receiver=0)  # MAIN: 144.1 MHz
await radio.set_frequency(144_200_000, receiver=1)  # SUB:  144.2 MHz

# Dual audio RX
radio.start_audio_rx(callback=on_main, receiver=0)
radio.start_audio_rx(callback=on_sub, receiver=1)
```

Comparison:
| Radio | Receivers | LAN | Serial |
|-------|-----------|-----|--------|
| IC-7610 | 2 | ✅ | ✅ |
| IC-705 | 1 | ✅ | ✅ |
| IC-7300 | 1 | ❌ | ✅ |
| IC-9700 | **2** | ✅ | ✅ |

- Profile-driven capability gating:
  - IC-705: 36 capabilities enabled
  - IC-7300: 36 capabilities enabled
  - IC-9700: 36 capabilities enabled (includes dual-receiver commands)

### Backend Factory Routing

**Serial Routing Logic:**
```python
model = (config.model or "IC-7610").upper()
if model == "IC-705":
    backend = Ic705SerialRadio
elif model == "IC-7300":
    backend = Ic7300SerialRadio
elif model == "IC-9700":
    backend = Ic9700SerialRadio
else:
    backend = Icom7610SerialRadio  # default
```

**LAN Routing Logic:**
```python
# LAN uses IcomRadio with model parameter
# model parameter triggers profile loading via resolve_radio_profile()
radio = IcomRadio(
    host="192.168.1.100",
    model="IC-9700"  # auto-detects CI-V 0xA2, dual-receiver capability
)
```

## Migration Guide

### From IC-7610 Only → Multi-Radio

**No breaking changes.** Existing IC-7610 code works unchanged:

```python
# Old code (still works)
radio = IcomRadio(backend="serial", device="/dev/cu.usbserial-123")

# New code (explicit model)
radio = IcomRadio(
    backend="serial",
    model="IC-7610",  # optional, defaults to IC-7610
    device="/dev/cu.usbserial-123"
)

# New: IC-705
radio = IcomRadio(
    backend="serial",
    model="IC-705",
    device="/dev/cu.usbserial-456"
)

# New: IC-9700 with dual receiver
freq_main = await radio.get_frequency(receiver=0)  # MAIN
freq_sub = await radio.get_frequency(receiver=1)   # SUB
```

### CLI Changes

```bash
# Old (IC-7610 only)
icom-lan --backend serial --device /dev/cu.usbserial-XXX freq

# New: Specify model explicitly
icom-lan --backend serial --model IC-7300 --device /dev/cu.usbserial-XXX freq
icom-lan --backend serial --model IC-9700 --device /dev/cu.usbserial-XXX freq

# New: Dual-receiver operations on IC-9700
icom-lan --backend serial --model IC-9700 --device /dev/cu.usbserial-XXX \
    --receiver 1 freq 144200000
```

## Known Limitations

### Hardware Blocked
- IC-705, IC-7300, IC-9700 backends are **software-ready** but **hardware validation pending**
- No physical radios available yet for testing
- All functionality is mocked in unit/contract tests

### Audio Codec
- **ulaw→pcm decoding incomplete** in web handlers
  - Current: Maps ULAW codecs to PCM16 without conversion
  - Planned: Proper ulaw decoder in v0.12.0
  - Workaround: Use Opus codec for web audio (recommended)

### Command 29 (Receiver Routing)
- IC-7300: No Command 29 support (hardware limitation)
- IC-9700: Has Command 29 for dual-receiver routing (supported)
