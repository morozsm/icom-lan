    IC-7610 Capability Coverage Audit¶

    Date: 2026-03-29 Scope: All capabilities defined in rigs/ic7610.toml vs implementation across 3 layers

    Summary¶

    Layer Implemented Total Coverage
    Python API (radio.py) ~210 ~216 97%
    Web API (WebSocket cmds) ~150 ~216 ~70%
    Rigctld (hamlib compat) 19 ~216 ~9%

    Gaps by Layer¶

    Python API — Missing (~6 commands)¶

    Command CI-V Notes
    get_scope_wave 0x27 0x00 Waveform type query
    set_scope_wave 0x27 0x00 Set waveform type
    scope_data_output 0x27 0x11 Handled via binary stream, not method
    get_main_sub_band 0x07 0xD2 Query which band is MAIN vs SUB
    get_main_sub_prefix 0x29 Low-level CI-V prefix — used internally
    set_main_sub_prefix 0x29 Low-level CI-V prefix — used internally

    Assessment: Effectively 100% — missing items are either internal protocol details (0x29) or handled differently (scope data = binary stream).

    Web API — Missing (~65 commands)¶

    Most missing items are menu settings (0x1A 0x05...) and low-frequency controls:

    TX Controls (no Web handler for SET): - set_vox (toggle VOX) - set_vox_gain, set_anti_vox_gain - set_vox_delay - set_drive_gain - set_break_in_delay - set_repeater_tone, set_repeater_tsql - set_tsql_freq - set_tuner_status, set_tuning_step - set_af_mute

    Data Mode / Modulation Inputs (all 6 get/set pairs): - get/set_data_off_mod_input - get/set_data1_mod_input, data2, data3 - get_acc1_mod_level, get_usb_mod_level, get_lan_mod_level

    DSP Advanced (no Web handler): - get/set_nb_depth, get/set_nb_width - get/set_manual_notch_width - set_audio_peak_filter - get/set_digisel_shift

    System/Config: - get_system_date, get_system_time, get_utc_offset - get/set_ref_adjust, get/set_civ_transceive - get/set_civ_output_ant - get/set_dash_ratio

    Meters (read-only, low priority): - get_s_meter_sql_status, get_various_squelch - get_overflow_status, get_vd_meter - get_speech

    Memory (newly added, API only): - set_memory_mode, memory_write, memory_to_vfo, memory_clear - set_memory_contents, get_memory_contents, get_memory_mode - get/set_bsr

    Band/Split: - get_split (read) - get_band_edge_freq - get/set_xfc_status, set_tx_freq_monitor - get/set_quick_split, get/set_quick_dual_watch

    Scan: - Scan start/stop exist ✅

    Recommendations¶

    Priority 1 — Web API gaps (user-facing)¶

    1. VOX controls — toggle, gain, delay (common in operation)
    2. CW text send/stop — needed for CW operation from browser
    3. Tone/TSQL toggles — needed for repeater use
    4. Modulation inputs — data mode config from browser

    Priority 2 — Rigctld expansion¶

    1. Levels — AF, RF, NR, NB, COMP, MIC, KEYSPD, CWPITCH (common hamlib queries)
    2. Functions — NB, NR, COMP, VOX, TONE, TSQL, ANF, LOCK, MON
    3. Raw passthrough (w) — allows any CI-V command via hamlib

    Priority 3 — Nice to have¶

    1. BSR via Web API
    2. Menu settings exposure (date/time, UTC, ref adjust)
    3. Advanced DSP params (NB depth/width, notch width)
